### RSACng and DSACng are once again usable in Partial Trust scenarios

|   |   |
|---|---|
|Details|CngLightup (used in several higher-level crypto apis, such as <code>T:System.Security.Cryptography.Xml.EncryptedXml</code>) and <code>T:System.Security.Cryptography.RSACng</code> in some cases rely on full trust. These include P/Invokes without asserting <code>F:System.Security.Permissions.SecurityPermissionFlag.UnmanagedCode</code> permissions, and code paths where <code>T:System.Security.Cryptography.CngKey</code> has permission demands for <code>F:System.Security.Permissions.SecurityPermissionFlag.UnmanagedCode</code>. Starting with the .NET Framework 4.6.2, CngLightup was used to switch to <code>T:System.Security.Cryptography.RSACng</code> wherever possible. As a result, partial trust apps that successfully used <code>T:System.Security.Cryptography.Xml.EncryptedXml</code> began to fail and throw <code>T:System.Security.SecurityException</code> exceptions.This change adds the required asserts so that all functions using CngLightup have the required permissions.|
|Suggestion|If this change in the .NET Framework 4.6.2 has negatively impacted your partial trust apps, upgrade to the .NET Framework 4.7.1.|
